Git commands cheat sheat


  • git -version : It provides us with the correct version of git installed in our systems. If not installed, it will give an error.

  • git init : This command will intialize an empty repository inside the current working directory in our system.

  • git status : status command provides us the current status of the files in the working directory if the files are ready to be staged or not.

  • git add file_name : add commands stages the selected file that has to be commited into the working directory. To stage all the files at once, use ' git add '..

  • git commit -m 'message' : this command is used to commit or basically, register the staged files into the working directory. It logs all the staged files.

  • git log : log provides all the commit history along with the commit hashcode which helps us to get back to previous versions.

  • git reset : reset command is used to go back to the previous versions of the code along with the hashcode provided in the log command. It is combined with the line arguments : --hard: to reset the code along with the version & --soft: to simple go back to previous version without resetting the code.

  • git remote add origin link : this command is used to attach/connect to an existing github repository where all the nescessary files and folders can be uploaded and projects can be hosted. 'link' refers to the github repository link.

  • git branch -m main : this command logs onto the main branch in the working directory.

  • git branch : branch command breifs about the active branches present in the working directory.

  • git branch branch_name : creates a new branch into the working directory.

  • git checkout branch_name : used to navigate between the active branches present in the working directory.

  • git merge branch_name : this command merges the specified branch with the main branch.

  • git push -u origin main : this command pushes the staged files to the main branch of the linked github repository.

  • git push : pushes the staged files and updates the main branch of the linked github repository.

  • git pull : this command extracts all the files present onto the repository in github.

  • git pull : this command extracts all the files present onto the repository in github.

  • git clone : clone command can be used to create a copy of an existing git repository.
